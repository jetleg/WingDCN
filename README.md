# WingDCN
WingDCN的仿真源码（数值分析层面）
Main.java是将网络拥塞问题建模成需求矩阵的约束
Latency.java求解的是，在存在拥塞的情况下，假如每一轮流量只有一部分比例可以成功获得路由分配（比如load=1.0时，这个比例大概是83%，
这个比值跟流量的（输入端口，输出端口）二元组分布有关，流量越多，真实比值越接近这个值）；
假设这个比例为a，也就意味着1-a比例的流量只能和下一轮新到来的流量竞争路由分配(实际上我们会给等待轮数更多的流量设置更高的优先度
，不过这并不会影响我们要求得的平均轮数的结果)，不考虑缓存队列满了的情况（不考虑丢包），那么对于这部分流量来讲，它们最终都会获得
路由分配，要求他们成功获得路由分配的平均等待轮数？（也就是如果它第一次参与分配就获得了端口，那它相当于等待了1轮，如果第二次才获得，
那就是2轮，求他们平均等了多少轮）

这个问题很像延毕，假如每年有一定量的学生入学，每年在读的这部分学生只有一定比例的学生可以毕业，那么每年都会有一部分学生延毕，求学生
毕业平均延毕时长。
